# 如何透過 AI 評估現有網站

在進行網站改版或優化前，了解現有網站的痛點是至關重要的第一步。傳統的 UI/UX 評估（例如：使用者測試、焦點小組、人工啟發式評估）通常需要耗費大量的時間與資源。

如今，透過**多模態 AI（Multimodal AI）**的協助，專案管理員與設計師可以在幾分鐘內，對現有網站進行客觀、全面且符合現代設計規範的 UI/UX 檢測。

---

## 💡 為什麼要用 AI 評估網站 UI/UX？

> [!NOTE]
> AI 不是要取代專業的設計決策，而是扮演**「超高效的客製化設計顧問」**，提供客觀、基於數據與業界標準的優化建議。

> [!IMPORTANT]
> **建立正確的評估心態：UI/UX 的「權衡與妥協 (Trade-offs)」**
> 在實際網頁設計中，**沒有任何一個網站能做到所有維度皆完美無缺**。每個頁面都有其「首要的核心目標」（例如：快速下單、瀏覽大量資訊、引人入勝的視覺衝擊），為了最大化這個核心重點，往往必須在其他面向做出適度的妥協：
> * **功能性與極簡的權衡**：一個功能強大且需要密集操作的專業系統（如票務後台、報稅系統），為了操作連貫性與高效率，視覺排版必然較為緊湊，無法追求極致的留白與呼吸感。
> * **美觀度與效能的權衡**：一個主打政令緊急通報或高載量公共服務的網頁，為了極限頻寬下的載入速度與無障礙相容性，必須捨棄華麗的過場動畫或高清影像，轉而使用純文字與精簡樣式。
>
> 因此，引導學生在看 AI 評估報告時，**切忌盲目追求「完美滿分」**。優秀的管理員必須結合「網站的核心任務」，客觀判斷哪些缺點是「為了解決核心問題而做的合理權衡（Trade-offs）」，哪些才是「真正阻礙使用者的致命痛點」。

| 優勢維度 | 說明 |
| :--- | :--- |
| **客觀啟發式分析** | AI 可以基於雅各·尼爾森（Jakob Nielsen）提出的 [10 大網頁可用性原則 (10 Usability Heuristics)](https://www.nngroup.com/articles/ten-usability-heuristics/)，逐一檢查網頁的系統狀態、一致性與容錯設計等細節，避免人為盲點。 |
| **多模態視覺判讀** | 現代 AI（如 Claude 3.5 Sonnet, GPT-4o）能直接「看懂」網頁截圖，判斷色彩對比度、排版層級、視覺焦距。 |
| **程式碼結構優化** | 可以將前端 HTML/CSS 提供給 AI，檢查語意化標籤（Semantic HTML）與無障礙網頁規範（A11y）。 |
| **快速產生改版建議** | 不僅指出問題，還能立即產出修改方針、新版的 Prompt，甚至直接生成重構後的頁面程式碼。 |

## 📖 網站評估指南目錄

請點擊下方章節進行深度學習與 Prompt 複製：

### [第一章 📸 方法一：多模態截圖分析](./01_多模態截圖分析.md)
* **適用場景**：已擁有網頁視覺畫面。
* **評估載體**：使用線上 AI（如 Claude.ai, ChatGPT, Gemini）上傳電腦與手機版長截圖。
* **核心 Prompts**：收錄 UI/UX 啟發式評估、視覺美學審查、操作動線流暢度 Prompt 模板。
* **操作指引**：Chrome / Edge 瀏覽器內建整頁長截圖詳細步驟。

### [第二章 🌐 方法二：提供 URL 分析](./02_提供URL分析.md)
* **適用場景**：快速對網址內容與文案進行黑箱評估。
* **評估載體**：使用具聯網功能之線上 AI（如 Gemini、Grok）輸入目標網址或複製貼上純文字。
* **核心 Prompts**：收錄資訊架構與文案語意、目標受眾 (TA) 內容契合度評估 Prompt 模板。
* **觀念釐清**：解說「方法二（黑箱/讀者）」與「方法三（白箱/工程師）」評估視角與比喻的差別。

### [第三章 🔌 方法三：使用 MCP 連接器審查結構](./03_使用MCP連接器審查結構.md)
* **適用場景**：深度技術指標（DOM 樹、無障礙規範、SEO 代碼、資訊安全）評估。
* **評估載體**：使用 `claude.ai` 網頁版 Connectors (Control Chrome) 連線本地瀏覽器分頁。
* **核心 Prompts**：收錄 ♿ 無障礙網頁評估 (Accessibility / A11y)、🔍 搜尋引擎優化 (SEO)、🔒 網站資訊安全 (Security) Prompt 模板。
* **原理介紹**：介紹什麼是 DevTools MCP，以及只給網址時 AI 爬蟲的侷限性。

## 🚀 評估後的下一步：如何快速重建原型？

當 AI 幫您指出網站的缺點並給出具體建議後，您可以利用以下步驟，**無縫過渡到原型設計**：

1. **萃取評估報告**：
   將 AI 給您的優化建議，轉化為具體的設計特徵。例如：「將首頁改為雙欄佈局，左側為焦點新聞，右側為常用申辦服務捷徑；色彩主色調改為深藍色，搭配暖橘色作為強調色。」
2. **生成改版 Prompt**：
   將這些特徵寫入 `ai輔助/README.md` 中介紹的【進階版 Prompt 模板】。
3. **送入原型生成工具**：
   將 Prompt 輸入 **Framer AI**、**v0.dev** 或 **Uizard**，瞬間即可生成新版的網站視覺原型與 HTML/CSS 程式碼。

> [!TIP]
> 善用這個「評估 ➡️ 產生建議 ➡️ 寫成 Prompt ➡️ 快速生成原型」的黃金循環，您能在一天內完成過去需要兩週溝通才能得出的改版初稿！
