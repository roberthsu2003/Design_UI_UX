# AI輔助

AI 驅動的 UI/UX 設計工具。

#### 1. Uizard
#### 2. Framer AI
* **強項：** 非常適合從零開始**透過文字提示生成高保真（High-fidelity）、響應式**的網站。
* **如何使用：** 您需要先「消化」您的 UI/UX 報告，然後用**描述性**的文字告訴 Framer AI 您想要的新設計。
    * *範例提示：* 「幫我設計一個電商網站的首頁，它需要有極簡的導覽列、一個置中的大型產品圖片，以及三欄式的特色介紹。」
* **優點：** 它產出的原型非常精緻，幾乎可以直接發布。

#### 3. Visily.ai
* **強項：** 與 Uizard 類似，它也擁有強大的**「截圖轉設計」**功能。
* **如何使用：** 工作流程與 Uizard 幾乎相同。您可以上傳現有網站的截圖，它會幫您轉換成可編輯的組件。
* **優點：** 它在識別設計組件（Components）方面做得很好，並且提供許多預設的主題和模板來加速修改過程。

### ➡️網頁模板

#### 模板 1：簡易版 (快速發想)

這個模板的目的是讓學生「克服空白頁的恐懼」。它專注於最核心的兩個問題：**「做什麼」和「感覺如何」**。


> 理解 AI 可以快速生成視覺原型 (Prototype)。重點在於快速迭代，先求有，再求好。

```
# 網頁設計請求 (簡易版)

## 1. 網頁主題
[請填入網頁的主題或目的。例如：一家賣手工餅乾的甜點店、一個個人旅遊部落格、一款新的筆記 App]

## 2. 視覺風格
[請填入 2-3 個描述感覺的關鍵字。例如：簡約、溫暖、專業、科技感、有趣、黑暗模式]

## 3. 你的任務
請根據上述主題和風格，幫我設計一個單頁式網站的首頁 (Homepage) 概念。
```

### 模板 2：中階版 (結構導向)

這個模板思考\*\*「網頁的結構」\*\*。一個網頁不只是一張圖，它有不同的區塊 (Sections)，每個區塊都有其特定的功能。


> 介紹「網頁佈局」(Layout) 的基本概念，如頁首 (Header)、主要內容 (Main Content) 和頁尾 (Footer)。並開始思考「使用者動線」和「行動呼籲 (CTA)」。

```
# 網頁設計請求 (中階版)

## 1. 網頁類型
[請填入網頁的具體類型。例如：產品介紹頁 (Landing Page)、公司形象官網、活動報名頁、個人作品集]

## 2. 網頁主題
[請填入產品或服務的名稱。例如：AI 語言學習 App "LingoBot"]

## 3. 網頁結構
請幫我設計一個網頁，它必須包含以下幾個區塊：

* **頁首 (Header)：** [需要放什麼？ 例如：Logo、導覽選單 (首頁, 關於我們, 價格)]
* **主要內容 (Main Content)：** [描述 2-3 個主要區塊。 例如：1. 一個大的歡迎標語和圖片。 2. 介紹 3 個主要功能。 3. 使用者評價。]
* **頁尾 (Footer)：** [需要放什麼？ 例如：聯絡資訊、社群媒體連結]

## 4. 主要行動呼籲 (Call to Action - CTA)
[希望訪客在這個頁面完成的最主要的一件事。 例如：點擊「免費試用」按鈕、留下 Email、立刻購買]

## 5. 你的任務
請根據上述規劃，設計這個網頁的佈局和視覺風格。
```

### 模板 3：進階版 (使用者中心導向)

這個模板是前兩個的綜合，並加入了 UI/UX 最核心的概念：**「扮演角色 (Persona)」和「目標受眾 (Target Audience)」**。它要求 AI 站在特定使用者的角度去思考設計。

> 這是 UI/UX 的核心。設計不是為了「好看」，而是為了「解決特定使用者的特定問題」。引導他們思考：**「我是誰？」**、**「我的訪客是誰？」**、**「我希望他們做什麼？」**

```
# 網頁設計請求 (進階版 / UIUX 導向)

## 1. 角色扮演 (Your Persona)
* **你的角色：** 你是一位資深的 UI/UX 設計師。
* **你的任務：** 為我設計一個高轉換率、使用者體驗流暢的網頁，並說明你的設計決策。

## 2. 產品情境 (Context)
* **產品/服務名稱：** [例如：寵物保姆預約平台 "PetPal"]
* **網頁的主要目標：** [訪客完成的最重要的一件事。 例如：讓飼主快速找到並預約一位保姆]

## 3. 目標受眾 (Target Audience)
* **他們是誰？** [描述你的理想客戶。 例如：25-45 歲的上班族，養狗或養貓，工作忙碌，需要值得信賴的人照顧寵物]
* **他們的需求/痛點：** [他們在找什麼？ 例如：害怕找到不專業的保姆、預約流程很麻煩、價格不透明]

## 4. 內容與結構 (Structure & Content)
請規劃以下區塊：
* **Hero 區塊 (頁首橫幅)：** 必須包含一個能打中「目標受眾」痛點的標題、副標題，和一個主要的 CTA 按鈕。
* **信任建立 (Trust Building)：** [如何讓訪客信任你？ 例如：顯示「保姆認證流程」、「真實用戶評價」]
* **功能/服務介紹 (Features)：** [介紹 3 個核心功能，並說明它們如何解決受眾的痛點。 例如：「AI 智能配對」、「24小時視訊監控」]
* **價格方案 (Pricing)：** [如果適用的話]
* **最後的 CTA 區塊：** [在頁尾前再次呼籲行動]

## 5. 視覺與語氣 (Style & Tone)
* **品牌語氣 (Tone of Voice)：** [例如：專業、令人信賴、友善、溫暖]
* **視覺風格與色調：** [例如：使用溫暖的大地色系、插畫風格、專業的真實照片]
```
