# 建立提案用的原型網頁：功能導向協作手冊 🚀

在進行網站提案與改版時，網站管理者（Project Manager）的核心挑戰在於**如何將業務想法轉化為可視化的介面，並在設計師與工程師之間建立高效、精準的溝通橋樑**。傳統以「工具為導向」（偏重單一工具的操作）的溝通流程，容易導致設計與技術脫節，增加溝通成本。

本指南以**網站管理者**的視角出發，將工作流重新梳理為**三個功能導向的核心主題**，並為每個主題提供獨立的深入指南與範例。我們將整合 **Google Stitch**（氛圍探索）、**Claude Design**（組件微調）與 **Figma**（設計定稿），協助您快速生成原型，並向設計師與工程師精確傳達需求。

## 🎯 三大核心主題目錄

請點擊下方主題連結，進入對應的獨立指南，每個主題皆預留了兩個具體實作範例：

### 1. [主題一 🎨：現有網站無設計稿，如何導入 Google Stitch 進行視覺與氛圍探索？](./01_無設計稿的氛圍與視覺探索.md)
*   **適用場景**：只有舊網站 URL 或長截圖，需要快速發想全新設計氛圍並展示互動流。
*   **核心工具**：Google Stitch（氛圍設計與自動畫面串接）。
*   **溝通交付**：提供視覺氛圍看板給設計師，提供點擊路徑示意給工程師。

### 2. [主題二 💬：針對現有一頁網頁，如何調整與優化其頁面結構和內容？](./02_修改單頁網頁的結構與內容.md)
*   **適用場景**：既有單頁內容雜亂或結構不佳，需要精準微調欄位佈局與行銷文案。
*   **核心工具**：Claude Design（對話式重構區塊與微調）。
*   **溝通交付**：提供真實文案原型給設計師對接設計系統，提供 HTML/CSS 參考代碼給工程師。

### 3. [主題三 🚀：如何從零到一建立全新的網頁頁面？](./03_建立全新網頁頁面.md)
*   **適用場景**：新增全新功能頁（如註冊表單或 FAQ），需要定義導覽層階與無縫體驗。
*   **核心工具**：Claude Design、Google Stitch 與 Figma 的整合工作流。
*   **溝通交付**：提供原型動線給設計師，在 Figma 中啟用 Dev Mode 提供精確變數給工程師。

## 💡 網站管理者必備：AI 原型設計的 UI/UX 檢核清單

在審查原型或引導設計師與工程師時，管理者請利用以下原則進行審核，確保網站兼具美感與易用性：

1.  **立體空間感是否清晰？**
    *   使用者點擊後，新畫面是「垂直滾動展開」（垂直）、「橫向滑出」（水平），還是「彈出覆蓋」（前後關係）？確保透過視覺動畫表達清楚。（參考：[讓頁面的前後關係視覺化](../垂直_水平_前後的立體構造/README.md#04-讓頁面的前後關系視覺化)）
2.  **使用者是否會迷路？**
    *   新頁面是否有清晰的導覽列、麵包屑或返回按鈕？確保使用者在任何時候都能「掌握身在何處的元素」。（參考：[掌握身在何處的四個元素](../垂直_水平_前後的立體構造/README.md#03-掌握身在何處的四個元素)）
3.  **是否做到了「無縫式體驗」？**
    *   對於短表單、提示訊息或局部更新，是否優先採用覆蓋式、嵌入式或增量式介面，以減少頁面重新載入的跳轉感？（參考：[無縫式介面](../無縫式介面/README.md)）
4.  **視覺與操作是否保持一致？**
    *   新頁面的按鈕大小、字體階層、主色調是否與既有網站相符？避免因風格分裂導致使用者產生不信任感。（參考：[一致性的效果](../一致性/README.md)）
