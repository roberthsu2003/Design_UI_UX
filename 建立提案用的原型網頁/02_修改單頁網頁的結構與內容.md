# 主題二：針對現有一頁網頁，如何調整與優化其頁面結構和內容？ 💬

現有的單頁網頁（如活動 Landing Page、產品宣傳單頁）往往會面臨資訊雜亂、結構不良或內容不合時宜的問題。身為管理者，您需要精確地調整網頁版面與內容，並在設計師與工程師之間傳遞標準化的重構方向。

本單元介紹如何結合 **Claude Design** 的對話式微調功能，快速重構頁面區塊，並導出符合規範的設計元素與前端程式碼。

---

## 💡 功能導向解決方案：利用 Claude Design 進行對話式重構

### 步驟 A：評估現有網頁結構與痛點
在著手修改前，管理者應結合網站架構的核心觀念，檢視該單頁的以下維度：
*   **版佈局型**：現有版型是單欄還是多欄？是否需要加入格狀設計？（參考：[佈局模式](../佈局模式/README.md)）
*   **立體構造**：資訊點擊後的展開方式是否清晰？（參考：[垂直、水平、前後的立體構造](../垂直_水平_前後的立體構造/README.md)）

### 步驟 B：使用 Claude Design 重新設計區塊
1. **載入程式碼或截圖**：將現有頁面的 HTML 程式碼或截圖上傳至 Claude Design。
2. **下達結構重構指令**：
    *   *Prompt 範例*：「*幫我重構這個首頁的 Hero Section。將原本緊湊的單欄佈局改為『雙欄式佈局』，左側放置搶眼的品牌標題與綠色 CTA 按鈕，右側放置帶有微幅懸停 hover 效果的產品卡片，並確保有足夠的呼吸感留白。*」

### 步驟 C：真實文案替換與細微控制
1. **填入真實文案**：直接在 Claude Design 中將範本文字修改為您的實際行銷文案，確保視覺長度與真實排版一致。
2. **利用控制項微調**：使用內建的調整滑桿，手動微調元素間距（Padding/Margin）、主色調對比與字體大小，直到畫面達到最佳視覺階層。
3. **導出資源**：導出該重構區塊的 SVG 向量檔或語意化 HTML/CSS 程式碼。

## 👥 與團隊的高效溝通指南

*   🗣️ **與設計師溝通（結構定稿與設計系統對接）**：
    *   **討論焦點**：將 Claude Design 產出的高保真區塊導入 Figma。與設計師溝通：「這是我用真實文案調整出來的雙欄結構與資訊階層。請幫忙納入我們目前的設計系統，微調間距並確保字體層級的一致性。」
    *   **交付物**：Claude Design 產出的區塊 SVG 或 HTML 原型。
*   🗣️ **與工程師溝通（前端程式碼與佈局對齊）**：
    *   **討論焦點**：展示 Claude Design 生成的組件。對工程師說：「這是調整後新結構的 HTML/CSS 語意化代碼。我們希望採用 RWD 響應式佈局，左欄與右欄在手機版上會自動堆疊為單欄，請以此結構進行重構開發。」
    *   **交付物**：Claude Design 導出的 HTML/CSS 程式碼參考。

## 📖 實作範例分享

以下提供兩個具體實作範例，展示如何對現有單頁進行結構重構：

### 範例一：[請在此填入範例一的名稱]
> *例如：優化現有產品 Landing Page 的 Hero Section*
*   **背景與痛點**：
    *   現有網頁：[連結或名稱]
    *   問題：Hero 區塊文字過多、CTA 按鈕不醒目，導致轉換率低下。
*   **Claude Design 重構過程**：
    1. 將 Hero 區塊的原始代碼貼上至 Claude Design。
    2. 下達指令：「*將單欄式版型重構為雙欄式，左側放吸引人的主標題與綠色大按鈕，右側使用圓角卡片展示產品概念圖，並增加 48px 的上下間距以創造呼吸感。*」
    3. 在預覽畫面中微調按鈕的 Hover 狀態，並導出程式碼。
*   **溝通成效**：
    *   設計師根據導出的 SVG 快速更新了 Figma 提案稿。
    *   工程師直接利用導出的 CSS 進行樣式替換，僅花 10 分鐘完成修改。

### 範例二：[請在此填入範例二的名稱]
> *例如：重構資訊過於擁擠的「聯絡我們」表單頁面*
*   **背景與痛點**：
    *   現有網頁：[連結或名稱]
    *   問題：聯絡表單有 10 個欄位，在同一個頁面中展開，導致行動裝置使用者在滾動時容易產生厭煩感。
*   **Claude Design 重構過程**：
    1. 將表單程式碼載入 Claude Design。
    2. 使用指令：「*將此長表單重構為『無縫步驟表單（Multi-step Form）』，第一步收集基本資訊（姓名、Email），第二步收集公司資訊與問題。上方加上進度條。*」
    3. 利用調整滑桿將輸入框的間距適度放大，改善點擊目標大小。
*   **溝通成效**：
    *   設計師同意將無縫步驟引入其他表單設計，以提升一致性。
    *   工程師依據產出的 HTML 結構實作前後端資料串接，避免使用者重複跳轉。
