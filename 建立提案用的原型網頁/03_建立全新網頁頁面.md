# 主題三：如何從零到一建立全新的網頁頁面？ 🚀

當專案需要新增一個全新功能的頁面（例如：會員註冊表單、常見問題互動頁面）時，身為網站管理者，您需要從零規劃它與既有網站的連結、定義頁面間的互動方式，並產出符合品牌一致性的原型供團隊開發。

本單元介紹如何結合 **Google Stitch**（互動串接）與 **Claude Design**（組件生成），並在 **Figma** 中定稿建立全新網頁頁面的全流程協作。

---

## 💡 功能導向解決方案：全流程 AI 輔助協作

### 步驟 A：規劃頁面導覽與立體構造
1. **防止使用者迷路**：新頁面必須具備清晰的定位元件（如麵包屑或全域導覽列）。（參考：[導覽列](../導覽列/README.md)）
2. **採用無縫體驗**：針對需要高轉換率或輕量互動的場景，優先考慮不切換畫面的 [無縫式介面](../無縫式介面/README.md)（如彈出式、滑動抽屜式），而非傳統的整頁跳轉。

### 步驟 B：利用 Claude Design/Stitch 快速生成頁面骨架
1. **下達功能性 UI 生成指令**：
    *   *Prompt 範例*：「*建立一個符合既有網站綠色簡約風格的『新會員註冊頁面』。使用『無縫式步驟（Multi-step Form）』，第一步填寫基本資料，第二步驗證信箱。表單上方需有進度條，並搭配柔和的綠色成功勾選動畫。*」
2. **利用 Stitch 設定點擊流程**：在 Stitch 中將新頁面與舊頁面的按鈕連結，確認整條 User Journey 的流暢度。

### 步驟 C：導入 Figma 定稿並進行元件化
1. **複用與擴充元件**：將生成的 UI 元素導入 Figma，把常見元件（如按鈕、輸入框）對齊現有的設計系統（Design System），確保網站的 [一致性](../一致性/README.md)。

---

## 👥 與團隊的高效溝通指南

*   🗣️ **與設計師溝通（無縫體驗與元件複用）**：
    *   **討論焦點**：展示新頁面的草圖，討論：「為了保持一致性，我們是否可以直接複用既有網站的導覽列與按鈕樣式？」、「這款無縫式步驟表單的微交互，應該如何在 Figma 中設定動畫過渡？」
    *   **交付物**：Stitch/Claude Design 的原型動線與 Figma 組件草案。
*   🗣️ **與工程師溝通（規格與技術實作）**：
    *   **討論焦點**：在 Figma 中啟用 Dev Mode。對工程師說：「這是新網頁的設計。我們複用了既有的按鈕與輸入框組件，只有表單進度條是新元件。請參考 Figma 的 CSS 屬性，並實作無縫步驟的切換邏輯，避免重新整理頁面。」
    *   **交付物**：Figma Dev Mode 規格標註連結與新組件的互動規格說明。

## 📖 實作範例分享

以下提供兩個具體實作範例，展示如何從零到一建立新網頁：

### 範例一：[請在此填入範例一的名稱]
> *例如：從零建立「無縫式會員註冊多步驟表單」頁面*
*   **背景與需求**：
    *   目標：新增一個會員註冊頁，希望流程簡單、防呆效果好，且流暢不跳轉。
*   **AI 輔助開發過程**：
    1. 在 Claude Design 中輸入多步驟註冊表單 Prompt。
    2. 生成包含進度條、即時驗證回饋（如密碼強度）的介面。
    3. 導入 Figma 連接按鈕，將按鈕元件與舊網站風格對齊。
*   **溝通成效**：
    *   與設計師確認了「無縫式步驟」的滑動轉場動畫（水平向右滑入）。
    *   與工程師討論了前端資料即時驗證的 API 對接點。

### 範例二：[請在此填入範例二的名稱]
> *例如：為現有網站新增「常見問題 FAQ 與互動摺疊面板」頁面*
*   **背景與需求**：
    *   目標：新增 FAQ 頁面，方便使用者自行搜尋並解決常見問題，降低客服成本。
*   **AI 輔助開發過程**：
    1. 使用 Claude Design 生成帶有互動式摺疊面板（Accordion）的 FAQ 頁面。
    2. 使用指令：「*建立一個 FAQ 頁面，左側為分類選單，右側為可折疊的常見問題卡片，點擊展開時有平滑過渡動畫。*」
    3. 使用內建微調控制項調整邊界線的色彩對比，確保閱讀舒適性。
*   **溝通成效**：
    *   與設計師定稿了分類選單的常駐與懸停（hover）樣式。
    *   與工程師確認利用原生 HTML `<details>` 與 `<summary>` 標籤或簡單的 CSS transition 實作摺疊動畫。
