# 🌐 方法二：提供 URL 分析

---

當你使用具備**聯網搜尋與抓取功能**的線上 AI（如 Gemini、Grok、ChatGPT 聯網版）時，直接向 AI 提供目標網頁的 **URL (網址)** 是一種極為快速的黑箱評估方法。AI 可以連線並擷取該網頁的文字內容與基礎架構，為你進行內容語意與資訊架構的初步診斷。

---

> [!TIP]
> **💡 學習重點：理解「方法二（提供 URL 分析）」與「方法三（使用 MCP 連接器）」的差別**
> 
> 雖然這兩種評估方式都不需要你上傳網頁圖片，但你所採取的**評估視角與目的**截然不同：
> * **方法二（提供 URL 分析）——「黑箱測試 / 讀者視角」**
>   * **你的操作**：直接提供網址，或將網頁上的「所有文字內容」全選複製貼給 AI。
>   * **你的評估重點**：看**內容文案與邏輯**。例如：網站的服務說明夠不夠白話？導覽選單的文字會不會讓人困惑？資訊階層的字彙是否符合一般人直覺？
>   * **生活化比喻**：就像讀者在看一本書的**「內容與目錄」**，檢查文章寫得好不好懂、章節順序合不合邏輯。
> 
> * **方法三（使用 MCP 連接器）——「白箱測試 / 工程師視角」**
>   * **你的操作**：在 Claude.ai 網頁版啟用連接器（如 Control Chrome 或 Kapture Browser Automation），讓 AI 直接連線至你的瀏覽器分頁。
>   * **你的評估重點**：看**底層骨架與技術規範**。讓 AI 直接讀取實時 DOM 樹與代碼結構，檢查如替代文字 alt 是否遺漏、標題層級是否正確、CSS 佈局是否有跑版風險，或是 JS 驗證邏輯是否安全。
>   * **生活化比喻**：就像修車師傅打開汽車的**「引擎蓋」**，檢查裡面的線路接得對不對、零件安不安全。

## 📝 實用 AI 評估 Prompt 模板 (適用於網址/文字)

你可以直接複製以下 Prompt 模板，貼入你欲評估的網址，發送給支援聯網的 AI：

### 1. 資訊架構與文案語意評估

> [!TIP]
> 🎯 **推薦實作範例：台灣自來水公司**
> 對於沒有經驗的學生，**「台灣自來水公司官網」是練習資訊架構與文案語意評估的最佳範例**。
> * **評估重點**：公用事業網站的導覽選單與最新消息常包含較多的官方硬口吻或繁雜分類，非常適合讓 AI 評估「這些文字對一般民眾而言是否白話易懂」以及「導覽結構分類是否符合直覺」。
> * **操作方式**：將台灣自來水公司官網網址 `https://www.water.gov.tw/` 貼入下方 Prompt 的網址欄位，直接發送給支援聯網的 AI 即可。

> **使用情境**：評估網頁文字內容的易讀性，以及導覽架構的語意邏輯（推薦使用：台灣自來水公司）。  
> **操作方式**：將以下 Prompt 與網址發送給聯網 AI。

```markdown
# 角色與任務
你是一位資深網站內容設計師 (Content Designer) 與資訊架構專家。請存取我提供的網址，並對其進行內容與資訊架構評估：

網址：[請在此處貼上你的網址]

# 評估重點
1. **文案易讀性**：網頁文字是否通順易懂？是否包含過多艱澀難懂的專用術語？
2. **語意結構**：導覽選單（全域與區域導覽列）的命名是否合乎一般人直覺？是否容易造成誤解？
3. **資訊階層**：各區塊的主副標題關係是否清晰明暸？使用者能否快速抓住重點？

# 輸出格式
- ❌ **語意與文案痛點**：列出最讓人困惑的文字或選單架構。
- ✍️ **文案與導覽改寫建議**：提供具體重構後的選單名稱與內容文案範本。
```

### 2. 目標受眾 (TA) 內容契合度分析

> [!TIP]
> 🎯 **推薦實作範例：臺北與我好好慢老（臺北市銀髮福利網）**
> 對於沒有經驗的學生，**「臺北與我好好慢老」官方網站是練習 TA 內容契合度評估的最完美案例**。
> * **評估重點**：此網頁的目標受眾 (TA) 是「高齡長輩」或是「需要幫長輩尋找福利的家屬」。長輩的核心痛點是「眼睛不好、理解複雜資訊較吃力、渴望被關懷」。學生可以讓 AI 評估目前的網頁文字是否足夠直覺、字體大小與對比在閱讀上是否友善，以及是否能改寫為更白話、更有同理心的語調。
> * **練習網址**：`https://elders.gov.taipei/`
> * **操作方式**：將上方網址貼入下方 Prompt 的網址欄位，直接發送給聯網 AI。

> **使用情境**：檢查網頁的寫作風格、語氣是否與網站的目標使用者匹配（推薦使用：臺北與我好好慢老）。  
> **操作方式**：將以下 Prompt 與網址發送給聯網 AI。

```markdown
# 任務
分析以下網址的內容，評估其是否有效傳達給目標受眾 (Target Audience)。

網址：[請在此處貼上你的網址]

# 請分析：
1. **TA 輪廓判斷**：從網頁目前的文案口吻與提供的資訊，判斷這個頁面主要是寫給誰看的？
2. **痛點解決度**：該網頁的內容是否有切中目標受眾的需求？核心資訊是否隱藏得太深？
3. **語氣與調性 (Tone & Voice)**：網頁語氣（如：官方冰冷、親切溫暖、專業科技等）是否合適？有何調整建議？
```
